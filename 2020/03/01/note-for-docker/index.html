    最近在把服务器上的应用尽量docker化，踩了不少坑，不过也正好学习docker和docker-compose的用法。

    此篇文章记录docker常用命令、使用方法，用于备忘。

    配置文件

    主要是dockerfile和docker-compose.yml两个文件，这俩的关系，简单的讲，dockerfile包含生成镜像的配置信息，docker-compose.yml包含启动容器的配置信息。

    Dockerfile

    用于生成docker镜像的配置文件，简单讲就是初始化镜像用的，比如拷贝文件到镜像、执行命令、暴露端口、设置用户等等。

    生成镜像的常用命令如下：

    shell

    docker build [OPTIONS] PATH | URL | -

    通过-t 指定镜像名/版本

    PATH为Dockerfile所在的目录

    格式

    以pwn_docker的dockerfile为例：

    dockerfile

    基本的书写格式就是：指令+操作

    使用dockerfile生成镜像，就是逐步执行dockerfile中的指令

    指令

    指令不多，列张表格好了：

    指令 操作(例子) 功能
    FROM ubuntu:16.04 指定初始镜像
    ARG CODE_VERSION=latest 定义变量
    WORKDIR 指定工作目录
    COPY COPY test /opt/test/ 拷贝文件
    RUN adduser taqini 执行系统命令
    CMD 执行命令
    EXPOSE 80 曝露端口
    ENV 定义环境变量
    ENTERPOINT 指定默认执行命令

    FROM

    指定初始镜像( Base Image )，写法有三种

    dockerfile

    一般直接FROM image:tag就行

    ARG

    用于定义变量，例如：

    dockerfile

    WORKDIR

    COPY

    dockerfile

    COPY的同时修改文件所有者：

    dockerfile

    RUN

    RUN就是执行系统命令，有两种格式：

    • RUN (shell form, the command is run in a shell, which by default is /bin/sh -c on Linux or cmd /S /C on Windows)

    • RUN ["executable", "param1", "param2"] (exec form)

    CMD

    CMD ["executable","param1","param2"] (exec form, this is the preferred form)

    CMD ["param1","param2"] (as default parameters to ENTRYPOINT)

    CMD command param1 param2 (shell form)

    EXPOSE

    配置暴露的端口

    dockerfile

    ENV

    环境变量

    ENTRYPOINT

    Docker-compose.yml

    主要用于启动docker容器的配置文件，yaml格式，简单讲就配置容器怎样启动，比如使用哪个镜像、怎样映射端口、如何挂载volume，执行什么命令等等，相当于docker run的参数配置文件。

    docker-compose命令根据Docker-compose.yml的内容生成镜像或管理容器(开启/停止/重启/查看/删除…)
